# Agent OS AI — 團隊上手指南

> 俾宏利保險顧問團隊用嘅內部工作台。呢份 guide 係畀新同事最快上手，同埋講清楚**私隱守則**（非常重要）。

---

## 1. 系統係咩？

Agent OS AI 係一個**純網頁**嘅保險顧問工作台，開個 browser 就用，唔使裝 app。核心係 6 個模組 + 1 個 AI 智能體（SET），幫你做內容、分析客戶、準備見客、寫 follow-up、出 proposal。

每位同事用**自己嘅 email 帳號**登入，資料喺雲端（Supabase）各自隔離——你睇唔到其他人嘅客戶資料。

---

## 2. 第一次使用：註冊 → 登入

1. 開個網頁連結（CloudStudio 分享 link）。
2. 第一次見到登入頁，撳 **「註冊新帳號」**，用你嘅**公司 email** 開個人帳號，密碼至少要 6 位。
3. Supabase 會 send 一封**確認信**去你個 email → 撳入面個 link 啟用帳號。
4. 返返登入頁，撳 **「登入」**。

**💡 小貼士：**
- 喺 email / 密碼欄撳 **Enter** 就會直接登入（唔使撳掣）。
- 如果見到**紅色 BAR**，唔使驚，佢係中文提示：
  - 「Email 或密碼錯誤」→ 檢查下打字
  - 「個 email 仲未確認」→ 去收確認信撳 link
  - 「網絡連唔到」→ 檢查網絡
  - 「已經註冊過」→ 直接登入就得

## 3. 六大模組一覽

左邊 sidebar 撳吓就轉頁：

| 模組 | 用嚟做咩 |
|------|----------|
| **工作台** | 總覽統計（客戶數、貼文數、follow-up、proposal） |
| **社交內容引擎** | 出小紅書 / 社交平台文案 + 配圖 prompt |
| **客戶分析** | 填客戶資料 → 自動計保障缺口、風險位 |
| **見客準備** | 生成見客 flow、反對處理話術 |
| **Follow-Up 生成器** | 自動寫 WhatsApp / email 跟進訊息 |
| **Proposal 引擎** | 出建議書框架 |
| **客戶列表 / 歷史記錄** | 睇返儲存咗嘅客戶同過往記錄 |

資料會自動儲存（本地 + 雲端同步），右上角有「💾 匯出備份」可以定期執份 JSON 防丟。

---

## 4. SET 智能體（AI 助手）— 點用

SET 係嵌入工作台嘅 AI 保險顧問，包晒六大範疇（VHIS、人壽危疾、ECI 勞工保險、MPF/TVC、Onboarding、內地客專項）。

**步驟：**
1. 左邊 sidebar 撳 **「SET 智能體」**（之前有 bug 開唔到，已經修好 ✅）。
2. 你會見到一列 agent 卡片（SET全能顧問、影子專家小Q、戰略規劃師、團體福利顧問、理財規劃師、客戶跟進助手）。撳其中一張就開到傾偈室。
3. 入面有**快捷 chip**（例如「分析客戶底牌」「VHIS 稅務扣減」），撳一下就發出去。
4. 底部輸入框打嘢，撳 Enter 發送。

**⚙️ 設定（右上角齒輪）：**
- **一鍵切 LLM 供應商**：SiliconFlow / DeepSeek / Gemini / 離線示範，撳一下即切，會保留你之前填嘅 API Key。
- **敏感模式（私隱守護）**：開咗之後，SET **絕對唔會將任何對話傳去雲端**，自動用離線示範回覆。

## 5. 🔒 私隱守則（全組必讀）

我哋會處理**真實客戶姓名、身份證、健康資料**，呢啲受香港《個人資料（私隱）條例》（PDPO）保障。請遵守：

1. **唔好直接貼真實客戶資料落 SET 對話**（姓名 / 身份證 / 電話 / 健康狀況）。
2. 如果一定要貼，先用輸入框左邊把 **🛡️ 遮蔽掣** —— 一撳就將身份證、電話、電郵、姓名、29 種疾病名自動替換成標籤。
3. 或者開 **敏感模式**（⚙️ 設定入面），咁對話完全唔出雲端，零外洩。
4. 記住：**SiliconFlow / DeepSeek 等中國 provider，數據會落中國**。涉及真實客戶資料時，優先開敏感模式，或者用本地/非中國 provider。
5. 雲端同步（Supabase）有 RLS 行級安全，每位同事只睇到自己嘅資料；但 sensible 資料仲係唔好亂貼。

> 簡單記住：**真名 + 健康資料 = 唔落 AI**。要分析就用遮蔽掣，或者敏感模式。

## 6. LLM 供應商點揀

SET 嘅 AI 回覆靠一個 LLM。可揀：

| 供應商 | 優點 | 注意 |
|--------|------|------|
| **SiliconFlow**（預設） | 中港通用、唔使 VPN、免費額度足 | 數據落中國，唔好貼真客資料 |
| **DeepSeek** | 香港平又快 | 數據落中國 |
| **Gemini（經 OpenRouter）** | 質素高 | 要 OpenRouter key |
| **離線示範** | 唔使 key、零外洩 | 回覆係預設模擬，唔係真 AI |

Key 去相應官網拎（SiliconFlow: cloud.siliconflow.cn；DeepSeek: platform.deepseek.com；OpenRouter: openrouter.ai），貼落 SET ⚙️ 設定就得。Key 只會存在你部瀏覽器 localStorage，唔會上傳任何 server。

## 7. 雲端同步 / 資料安全

- 登入後，客戶同歷史記錄會自動 sync 去 Supabase（你嘅帳號隔離）。
- 換部機登入同一個 email，資料就返晒嚟。
- 唔使擔心公司電腦壞咗 —— 定期撳「💾 匯出備份」執份 JSON 仲穩陣。

## 8. 常見問題

**Q: SET 開唔到？**
A: 已經修復（之前 router 漏咗 'set'）。如果仲開唔到，試下 refresh 個頁，或者 logout 再 login。

**Q: 登入撳 Enter 冇反應？**
A: 已經加咗 Enter 綁定。如果冇反應，檢查下個欄有冇 focus（撳吓 input 先），或者網絡連唔到。

**Q: 紅色 BAR 係咩？**
A: 登入錯誤提示，已經中文化。睇第 2 節解讀。

**Q: SET 回覆好似唔係真 AI？**
A: 可能你仲喺「離線示範」模式，或者未填 API Key。去 ⚙️ 設定一鍵切去 SiliconFlow 同填 key。

**Q: 我嘅客戶資料同事睇唔睇到？**
A: 睇唔到。Supabase RLS 行級安全，每人只睇自己。
